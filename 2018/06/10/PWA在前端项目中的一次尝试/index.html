什么是 PWA (官方文档)

Progressive Web App, 简称 PWA，是提升 Web App 的体验的一种新方法，能给用户原生应用的体验。

PWA 能做到原生应用的体验不是靠特指某一项技术，而是经过应用一些新技术进行改进，在安全、性能和体验三个方面都有很大提升，PWA 本质上是 Web App，借助一些新技术也具备了 Native App 的一些特性，兼具 Web App 和 Native App 的优点。
PWA 的主要特点包括下面三点：

  • 可靠 - 即使在不稳定的网络环境下，也能瞬间加载并展现
  • 体验 - 快速响应，并且有平滑的动画响应用户的操作
  • 粘性 - 像设备上的原生应用，具有沉浸式的用户体验，用户可以添加到桌面

创建本地缓存库

serviceWorker register之后，会调用install事件，install有个waitUntil事件，waitUntil执行完成之后，才进行安装。
在waitUntil里使用caches这个全局变量，使用它的open事件打开一个版本库，并将需要缓存的文件加入其中。

拦截请求

我们知道，无论是加载html js json 或者请求后台接口，本质上都是向指定资源发送http请求
fetch事件会在请求发送时触发，如果监测到要请求的资源已在版本库里缓存好了，就从缓存里读取。

版本控制

客户端创建了缓存库，但是发布新版本时，怎么通知客户端从服务器获取资源，而不是从缓存里取呢？

我们需要修改sw.js文件的版本库名称

我们在activate事件里遍历缓存库中的资源，监测版本库version和文件资源version不符，就把资源从版本库里delete掉，去服务器加载资源并更新缓存库
